# Workflow: Test-Only

## Steps

1. **classify_task** — Confirm task type is `test-only`
2. **discover_context** — Identify code under test and existing coverage
3. **identify_constraints** — Determine testing framework, conventions, and coverage targets
4. **select_skills** — Choose applicable skills for this test task
5. **generate_plan** — Design the test strategy
6. **write_tests** — Write failing tests that define the desired coverage (TDD red phase)
7. **define_tests** — Review test quality, add edge cases, confirm tests fail for the right reasons
8. **validate_solution** — Confirm tests are correct and meet coverage goals
