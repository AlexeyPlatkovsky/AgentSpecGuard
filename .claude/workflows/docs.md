# Workflow: Docs

## Steps

1. **classify_task** — Confirm task type is `docs`
2. **discover_context** — Identify what needs documentation and current state
3. **identify_constraints** — Determine documentation standards and scope
4. **select_skills** — Choose applicable skills for this docs task
5. **generate_plan** — Outline the documentation changes
6. **define_changes** — Write or update documentation
7. **define_tests** — Verify documentation accuracy and completeness
8. **validate_solution** — Confirm documentation is consistent and correct
