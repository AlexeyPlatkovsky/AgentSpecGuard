# Workflow: Investigation

## Steps

1. **classify_task** — Confirm task type is `investigation`
2. **discover_context** — Gather all relevant information, logs, and code paths
3. **identify_constraints** — Define scope boundaries and time/effort limits
4. **select_skills** — Choose applicable skills for this investigation
5. **generate_plan** — Structure the investigation approach
6. **define_changes** — Document findings and analysis
7. **define_tests** — Identify verification criteria for conclusions
8. **validate_solution** — Confirm findings are supported by evidence
