# Workflow: Feature

## Steps

1. **classify_task** — Confirm task type is `feature`
2. **discover_context** — Understand requirements, existing patterns, and integration points
3. **identify_constraints** — Determine API contracts, architectural boundaries, and dependencies
4. **select_skills** — Choose applicable skills for this feature
5. **generate_plan** — Design the implementation approach
6. **write_tests** — Write failing tests that define the expected behavior (TDD red phase)
7. **define_changes** — Implement the feature to make failing tests pass (TDD green phase)
8. **define_tests** — Harden tests, add edge cases, refactor tests
9. **validate_solution** — Verify the feature meets requirements and passes all tests
